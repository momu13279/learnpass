## 1.2 项目背景

随着高校信息化教学的不断推进，超星学习通（以下简称"学习通"）已成为国内高校广泛使用的在线教学平台之一。学习通承载着课程通知、作业布置、在线考试、成绩查询等核心教学功能，是高校学生日常学习中不可或缺的工具。然而，学习通平台本身在作业和考试管理方面存在诸多不足：不同课程的作业、考试、通知分散在各自的课程页面中，学生需要逐个课程查看，耗时且容易遗漏；缺乏统一的跨课程时间线视图，无法一目了然地了解近期所有待办任务的截止时间；缺少灵活的到期提醒机制，学生容易因疏忽而错过截止日期；互评任务与普通作业混在一起，状态跟踪困难。

为解决上述问题，本项目开发了一款名为"LearnPass 学习通助手"的桌面应用程序，采用 Electron + Vue 3 技术栈，通过对接学习通平台API，为高校学生提供一站式的作业与考试管理服务。系统能够自动同步学习通平台上的课程、作业、考试等数据，统一展示在一个界面中，并提供状态筛选、日历视图、提醒设置、数据统计等辅助功能，帮助学生高效管理学习任务，避免遗漏重要作业和考试。

# 第二章 项目概述

## 2.1 选题背景与意义

### 2.1.1 选题背景

在高校数字化教学转型的背景下，超星学习通平台已成为高校学生的日常学习工具。根据教育部统计数据，截至2024年，全国普通高等学校在校生规模超过4000万人，其中绝大多数学生都在使用学习通或类似的在线教学平台。使用学习通的高校学生平均每学期需管理8-15门课程，每门课程可能有多个作业和考试任务。

然而，学习通平台在作业和考试管理方面存在明显不足：信息分散在各课程页面中，缺乏统一时间线视图，无到期提醒机制，状态跟踪困难。这些问题导致学生经常错过作业截止日期或考试时间，影响学业成绩。在知乎、微博、B站等社交平台上，关于"学习通作业管理""错过作业截止日期"等话题的讨论热度持续走高，反映出学生群体对此类工具的强烈需求。

### 2.1.2 选题意义

本项目具有以下几方面的重要意义：

1. **实用价值**：切实解决高校学生在学习任务管理方面的实际痛点，帮助学生更好地掌握作业进度和考试安排，减少遗漏和逾期情况；
2. **技术锻炼**：通过完整经历软件工程的生命周期（需求分析、概要设计、详细设计、编码实现、测试），系统性地提升软件工程实践能力；
3. **技术应用**：将Electron跨平台桌面应用开发、Vue 3前端框架、API对接与数据解析等技术进行综合实践，加深对现代Web开发技术的理解；
4. **作品积累**：项目成果可作为个人技术作品集的重要组成部分，展示完整的软件开发能力，提升就业竞争力。

## 2.2 项目目标

本项目的核心目标如下：

1. **数据聚合**：自动同步学习通平台上的作业、考试、课程等数据，统一展示在一个界面中，消除信息分散问题；
2. **状态管理**：对作业进行分类管理（待完成、已逾期、已完成、待互评），帮助学生清晰掌握任务进度；
3. **时间管理**：提供日历视图，按日期展示作业和考试安排，支持状态颜色区分，帮助学生合理安排学习时间；
4. **智能提醒**：支持自定义作业和考试截止提醒，避免遗漏重要任务；
5. **数据统计**：提供作业和考试完成情况的可视化统计分析，辅助学生进行学情自查；
6. **桌面体验**：作为原生桌面应用，提供流畅、稳定的用户体验，支持系统托盘常驻。

# 第三章 软件工程各阶段工作概述

## 3.1 可行性分析阶段

### 3.1.1 工作内容概述

可行性分析阶段是项目开发的第一个阶段，主要工作是从技术可行性、市场需求可行性、经济可行性和社会可行性四个维度对项目进行全面论证，评估项目是否值得投入开发。

在本阶段，主要开展了以下工作：

1. **技术可行性分析**：调研并评估了Electron、Vue 3、Element Plus、Node.js、Axios等技术方案的成熟度和适用性，分析了API接口变更、Cookie过期与反爬机制、应用体积、跨平台兼容性、数据安全等技术风险，并为每个风险制定了应对措施；
2. **市场需求分析**：通过用户痛点分析，总结出信息分散、缺乏统一时间线、无到期提醒、状态跟踪困难、互评任务管理混乱等五大痛点；验证了全国高校学生对学习管理工具的强烈需求；分析了市场规模和增长趋势；
3. **竞争分析**：调研了学习通官方APP、网页版、浏览器插件和自建脚本等竞争产品，明确了本项目在桌面端原生体验、一站式数据聚合、智能提醒机制、数据统计分析和离线缓存支持方面的差异化优势；
4. **经济可行性分析**：估算了开发成本（人力、工具、设备、服务器）和运营成本（域名、签名证书、应用分发），确认了学生团队自主开发模式下资金投入极低的结论；
5. **社会可行性分析**：从教育信息化政策导向、学生心理健康需求和学习效率提升需求三个角度论证了项目的社会价值。

### 3.1.2 主要成果

本阶段产出了以下主要成果：

- 《LearnPass 学习通助手可行性分析报告》（docs/可行性分析报告.md），包含6个章节：概述、技术可行性分析、市场需求分析、经济可行性分析、社会可行性分析、结论。报告全文约270行，涵盖4张分析表格。

可行性分析的最终结论为：本项目在技术、市场、经济和社会四个维度均具有较高的可行性，建议立项开发。项目开发应分三个阶段推进：第一阶段（MVP）实现核心功能，第二阶段完善增强功能，第三阶段优化用户体验和探索产品化运营。

1. **软件工程流程的完整实践**：从可行性分析到测试验收，完整经历了软件工程的生命周期，深刻理解了各阶段的工作内容、输入输出和文档规范，认识到前期设计和文档编写对后续开发效率的重要影响；

2. **Electron跨平台开发能力**：掌握了Electron框架的多进程架构、IPC通信机制、安全隔离策略、窗口管理和Session管理等核心技术，理解了桌面应用与Web应用在架构上的差异；

3. **Vue 3前端开发能力**：深入学习了Vue 3的Composition API、响应式系统和计算属性，实践了Element Plus组件库的使用，提升了前端界面设计和交互实现能力；

4. **API对接与数据处理能力**：积累了与第三方平台API对接的实战经验，包括Cookie认证、HTTP请求管理、HTML数据解析、正则表达式应用和数据去重合并等技能；

5. **问题分析与解决能力**：在开发过程中遇到HTML解析、Cookie管理、数据去重等技术难题时，学会了通过分析问题根因、查阅官方文档和参考资料来寻找解决方案；

6. **文档编写能力**：通过编写5份项目文档，提升了技术文档的结构化表达能力和规范性，理解了GB/T 8567-2006等国家标准对软件文档编制的要求。

## 4.4 不足与改进方向

本项目在开发过程中也暴露了一些不足，可在后续版本中改进：

1. **自动化测试不足**：当前测试主要依赖手工测试，后续可引入单元测试框架（如Jest或Vitest）编写自动化测试用例，提升测试效率和回归测试覆盖率；

2. **日志系统简陋**：当前采用console输出日志，后续可引入electron-log等专业日志库，支持文件日志、日志分级和日志轮转，便于问题排查和运行监控；

3. **数据存储方案局限**：JSON文件存储在数据量增长后可能出现性能问题，后续可考虑迁移至SQLite嵌入式数据库，支持更高效的查询和索引；

4. **错误恢复机制不完善**：当前数据文件损坏时直接使用默认数据恢复，可能导致用户数据丢失，后续可增加数据备份机制和更完善的数据恢复策略；

5. **用户体验优化空间**：同步过程中缺少详细的进度百分比显示，日历视图中事件较多时显示拥挤，提醒设置的时间选择粒度较粗，这些细节可在后续版本中优化；

6. **多平台支持**：当前主要在Windows平台开发和测试，后续可增加macOS和Linux平台的兼容性测试和适配。
