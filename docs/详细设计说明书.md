# 学习通作业考试管理助手详细设计说明书

## 1. 引言

### 1.1 编写目的

本文档是"学习通作业考试管理助手"项目的详细设计说明书，旨在对系统的各模块、各组件的内部实现进行详细描述，为开发人员提供编码指导，为测试人员提供测试依据，为项目维护人员提供技术参考。

本文档的预期读者包括：
- **开发人员**：依据本文档中的接口定义、数据结构、流程描述进行编码实现
- **测试人员**：依据本文档中的功能设计编写测试用例
- **项目管理人员**：了解系统技术细节，评估开发进度
- **维护人员**：在系统上线后依据本文档进行问题排查和功能扩展

### 1.2 项目范围

本项目是一款基于 Electron 框架的桌面应用程序，面向高校学生用户，用于统一管理超星学习通平台上的作业与考试任务。系统通过 Cookie 认证方式与学习通平台交互，自动获取课程、作业、考试等数据，并提供日历视图、提醒设置、数据统计、数据导出等辅助功能。

**技术栈**：Electron 28 + Vue 3 + Element Plus + Node.js + Axios

**核心功能模块**：
- 本地用户登录与个人信息管理
- 学习通平台账号登录（Cookie 认证）
- 课程、作业、考试数据的自动同步与解析
- 作业管理（列表展示、状态筛选、详情查看、状态标记）
- 考试管理（列表展示、状态筛选、详情查看、状态标记）
- 日历视图（按月查看作业与考试安排）
- 提醒设置（作业/考试提前提醒时间配置）
- 数据统计（任务概览、完成情况统计）
- 数据导出（JSON 格式、文本格式）

主进程文件为 `main.js`，基于 Electron 的 `app`、`BrowserWindow`、`ipcMain`、`dialog`、`session` 模块构建，负责窗口生命周期管理、IPC 通信处理、学习通登录流程、数据持久化存储等核心功能。

### 2.2 IPC 处理器设计

主进程通过 `ipcMain.handle()` 注册一系列异步 IPC 处理器，供渲染进程通过预加载脚本调用。所有处理器返回统一格式的结果对象 `{ success: boolean, data?: any, message?: string }`。

#### 2.2.1 用户认证相关

| IPC Channel | 参数 | 返回值 | 说明 |
|-------------|------|--------|------|
| `login` | `{ username: string, password: string }` | `{ success, user?, message? }` | 本地用户登录验证 |
| `getUserProfile` | 无 | `{ success, data: UserProfile }` | 获取当前用户信息 |
| `updateUserProfile` | `{ id, name?, email?, phone? }` | `{ success, message? }` | 更新用户个人信息 |

**login 处理器逻辑**：
1. 在 `appData.users` 数组中查找匹配 `username` 和 `password` 的用户
2. 若找到，更新 `last_login` 字段为当前 ISO 时间戳，保存数据，返回用户信息
3. 若未找到，返回 `{ success: false, message: '用户名或密码错误' }`

#### 2.2.2 课程与作业管理

| IPC Channel | 参数 | 返回值 | 说明 |
|-------------|------|--------|------|
| `getCourses` | 无 | `{ success, data: Course[] }` | 获取全部课程列表 |
| `getHomeworks` | 无 | `{ success, data: Homework[] }` | 获取全部作业（含课程名） |
| `getHomeworkDetail` | `id: number` | `{ success, data: Homework?, message? }` | 获取单个作业详情 |
| `updateHomeworkStatus` | `{ id: number, status: string }` | `{ success, message? }` | 更新作业状态 |
| `getExams` | 无 | `{ success, data: Exam[] }` | 获取全部考试（含课程名） |
| `getExamDetail` | `id: number` | `{ success, data: Exam?, message? }` | 获取单个考试详情 |
| `updateExamStatus` | `{ id: number, status: string }` | `{ success, message? }` | 更新考试状态 |

**getHomeworks 处理器逻辑**：
1. 遍历 `appData.homeworks` 数组
2. 对每条作业，通过 `course_id` 在 `appData.courses` 中查找对应课程
3. 将课程名合并到作业对象中（优先使用作业自带的 `course_name`，其次使用关联课程的 `course_name`）
4. 返回合并后的完整作业列表

#### 2.2.3 学习通相关

| IPC Channel | 参数 | 返回值 | 说明 |
|-------------|------|--------|------|
| `openChaoxingLogin` | 无 | `{ success }` | 打开学习通登录窗口 |
| `getChaoxingStatus` | 无 | `{ success, data: ChaoxingStatus }` | 获取学习通连接状态 |
| `chaoxingLogout` | 无 | `{ success }` | 退出学习通登录 |
| `syncData` | 无 | `{ success, message? }` | 触发学习通数据同步 |

**getChaoxingStatus 处理器逻辑**：
返回学习通连接状态，其中 `cookie` 字段脱敏处理为 `'***'`，避免敏感信息泄露到渲染进程。

#### 2.2.4 提醒与设置

| IPC Channel | 参数 | 返回值 | 说明 |
|-------------|------|--------|------|
| `getReminders` | 无 | `{ success, data: Reminder[] }` | 获取全部提醒 |
| `createReminder` | `Reminder` | `{ success, data: Reminder }` | 创建新提醒 |
| `deleteReminder` | `id: number` | `{ success, message? }` | 删除提醒 |
| `getSettings` | 无 | `{ success, data: Settings }` | 获取应用设置 |
| `updateSettings` | `Settings` | `{ success }` | 更新应用设置 |

#### 2.2.5 搜索与导出

| IPC Channel | 参数 | 返回值 | 说明 |
|-------------|------|--------|------|
| `searchItems` | `query: string` | `{ success, data: { homeworks, exams } }` | 全文搜索 |
| `exportData` | `format: 'json' \| 'text'` | `{ success, data: ExportObject }` | 数据导出 |
| `getStats` | 无 | `{ success, data: StatsData }` | 获取统计数据 |

**searchItems 处理器逻辑**：
1. 将查询字符串转为小写
2. 在作业列表中搜索 `homework_name`、`course_name`、`description` 字段
3. 在考试列表中搜索 `exam_name`、`course_name`、`description`、`location` 字段
4. 返回匹配的作业和考试列表，每条记录附带 `type` 字段标识类型

**getStats 处理器逻辑**：
1. 计算作业总数、已完成数、已逾期数、待处理数
2. 计算考试总数、已完成数、待参加数
3. 计算本周任务数：获取本周起止时间（周日到周六），统计截止时间/开始时间在本周内的作业和考试数量

### 2.3 登录流程设计

#### 2.3.1 本地登录流程

```
用户输入用户名和密码
        |
        v
渲染进程调用 window.api.login({ username, password })
        |
        v
preload.js 通过 ipcRenderer.invoke('login', credentials) 转发
        |
        v
主进程 ipcMain.handle('login') 处理
        |
        v
在 appData.users 中查找匹配用户
        |
   +----+----+
   |         |
匹配成功    匹配失败
   |         |
   v         v
更新 last_login    返回 { success: false, message }
保存数据
   |
   v
返回 { success: true, user }
   |
   v
渲染进程设置 loggedIn = true，加载全部数据
```

#### 2.3.2 学习通登录流程

```
用户点击"登录学习通"按钮
        |
        v
渲染进程调用 window.api.openChaoxingLogin()
        |
        v
主进程调用 openChaoxingLogin()
        |
        v
创建 loginWindow，加载 https://passport2.chaoxing.com/login
        |
        v
页面加载完成后注入提示条（executeJavaScript）
        |
        v
用户在学习通页面完成登录
        |
        v
页面跳转到 i.chaoxing.com / mooc1.chaoxing.com / www.chaoxing.com
        |
        v
did-navigate / did-navigate-in-page 事件触发
        |
        v
调用 handleLoginSuccess(loginSession, url)
        |
        v
从 loginSession 获取所有 Cookie，拼接为 Cookie 字符串
        |
        v
保存到 appData.chaoxing.cookie
设置 appData.chaoxing.loggedIn = true
保存数据到磁盘
        |
        v
通过 mainWindow.webContents.send('chaoxing-login-success', data) 通知主窗口
        |
        v
关闭 loginWindow
```

**handleLoginSuccess 函数签名**：

```javascript
async function handleLoginSuccess(loginSession, url)
```

**Cookie 获取逻辑**：
1. 调用 `loginSession.cookies.get({ url: 'https://chaoxing.com' })` 获取所有 Cookie
2. 将 Cookie 数组拼接为 `name=value; name=value` 格式的字符串
3. 验证 Cookie 字符串长度大于 10（确保有效）
4. 保存到 `appData.chaoxing.cookie` 并持久化

### 2.4 数据存储设计

#### 2.4.1 存储路径

```javascript
const DATA_DIR = path.join(app.getPath('userData'), 'data');
const DATA_FILE = path.join(DATA_DIR, 'app_data.json');
```

数据文件存储在 Electron 的 `userData` 目录下的 `data/app_data.json` 文件中。`userData` 在 Windows 系统上通常为 `C:\Users\<用户名>\AppData\Roaming\learnpass-assistant\`。

#### 2.4.2 数据加载

```javascript
function loadData()
```

**加载流程**：
1. 调用 `ensureDataDir()` 确保数据目录存在（若不存在则递归创建）
2. 检查 `DATA_FILE` 是否存在
3. 若存在，读取文件内容并解析为 JSON 对象，赋值给全局变量 `appData`
4. 兼容性检查：确保 `appData.chaoxing` 和 `appData.currentId` 字段存在，若缺失则补充默认值
5. 若文件不存在，调用 `getDefaultData()` 创建默认数据并立即保存

#### 2.4.3 数据保存

```javascript
function saveData()
```

**保存流程**：
1. 调用 `ensureDataDir()` 确保目录存在
2. 将 `appData` 对象序列化为格式化的 JSON 字符串（缩进 2 空格）
3. 通过 `fs.writeFileSync()` 同步写入文件
4. 异常捕获：若写入失败，输出错误日志，不影响应用继续运行

#### 2.4.4 默认数据模板

```javascript
function getDefaultData()
```

返回包含以下结构的默认数据对象：
- `users`：包含一个默认本地用户（username: 'local', password: '123456', name: '本地用户'）
- `courses`：空数组
- `homeworks`：空数组
- `exams`：空数组
- `reminders`：空数组
- `settings`：默认设置对象（手动同步、浅色主题、中文、默认提醒时间）
- `currentId`：自增 ID 计数器（homework: 1, exam: 1, reminder: 1）
- `chaoxing`：学习通连接状态（未登录、无 Cookie、无同步记录）

### 3.1 接口暴露设计

### 3.2 安全隔离设计

系统采用多层安全隔离机制保护应用安全：

#### 3.2.1 上下文隔离（Context Isolation）

主窗口和登录窗口均配置 `contextIsolation: true`，确保渲染进程的 JavaScript 全局环境（`window` 对象）与主进程的 Node.js 环境完全隔离。渲染进程无法直接访问 `require`、`process`、`fs` 等 Node.js 模块。

#### 3.2.2 contextBridge 白名单机制

预加载脚本仅通过 `contextBridge.exposeInMainWorld('api', { ... })` 暴露明确列出的接口方法。渲染进程只能通过 `window.api.xxx()` 调用这些预定义的方法，无法访问未暴露的任何 Node.js 功能。

#### 3.2.3 Node.js 集成禁用

所有窗口配置 `nodeIntegration: false`，禁止渲染进程直接使用 Node.js API。远程模块（`enableRemoteModule: false`）同样被禁用，防止 `require('electron').remote` 等不安全调用。

#### 3.2.4 Cookie 安全处理

`getChaoxingStatus` IPC 处理器在返回数据时，将 Cookie 字段替换为 `'***'`，避免完整的 Cookie 字符串泄露到渲染进程。

#### 3.2.5 IPC 通信模式

所有 IPC 通信均采用 `ipcRenderer.invoke()` / `ipcMain.handle()` 的异步请求-响应模式，不使用 `send()` / `on()` 的单向消息模式（事件监听除外），确保每个调用都有明确的返回值。

## 4. 前端界面详细设计

前端界面基于 Vue 3 Composition API 构建，使用 Element Plus 组件库，所有代码包含在 `index.html` 单文件中。

### 4.1 页面结构设计

#### 4.1.1 整体架构

应用采用单页面架构（SPA），通过 `currentPage` 响应式变量控制当前显示的页面。页面切换通过 `v-if` 条件渲染实现。

**页面路由映射**：

| 页面标识 | 页面名称 | 说明 |
|----------|----------|------|
| `home` | 概览 | 首页仪表盘，展示任务统计和近期任务 |
| `homework` | 作业管理 | 作业列表、筛选、详情 |
| `exam` | 考试管理 | 考试列表、筛选、详情 |
| `calendar` | 日历视图 | 按月查看作业和考试安排 |
| `reminder` | 提醒设置 | 配置提醒时间和同步频率 |
| `profile` | 个人中心 | 个人信息、学习通连接、数据导出 |

#### 4.1.2 全局组件

### 5.5 状态判断逻辑设计

#### 5.5.1 作业状态判断

作业状态判断采用多层级策略：

```
第一层：从 HTML 中提取的状态文本关键词匹配
    |
    +-- "已批阅"/"已提交"/"已完成"/"待批阅"/"已互评" → completed
    +-- "待互评" → peer_review
    +-- "已截止"/"已过期"/"逾期" → overdue
    +-- "未交"/"未提交" → 进一步判断剩余时间
    |       |
    |       +-- 无剩余时间或包含"已过期"/"逾期" → overdue
    |       +-- 有剩余时间 → pending
    +-- "进行中" → in_progress
    +-- "待重做" → overdue
    |
第二层：检查内容文本中是否包含"已互评"/"待互评"（可能在其他位置）
    |
第三层：若仍为 pending，根据截止时间自动判断
    +-- 截止时间 < 当前时间 → overdue
```

#### 5.5.2 考试状态判断

```
第一层：从 ks_state 标签提取状态文本
    |
    +-- "已结束"/"已完成"/"待批阅"/"已批阅" → completed
    +-- "已过期"/"已截止" → overdue
    +-- "进行中" → in_progress
    |
第二层：若仍为 pending，根据结束时间自动判断
    +-- 结束时间 < 当前时间 → overdue
    |
第三层：若仍无结束时间，从标题中提取日期判断
    +-- 标题日期 < 当前日期 → overdue
```

### 5.6 数据同步流程设计

```javascript
async function syncAll(cookieStr)
```

**同步流程**：

```
syncAll(cookieStr)
        |
        v
validateCookie(cookieStr)  ——> Cookie 无效 ——> 返回失败
        |
        v Cookie 有效
Promise.all([
  getCourseList(cookieStr),     // 并行获取课程
  getAllHomeworks(cookieStr),   // 并行获取作业
  getAllExams(cookieStr)        // 并行获取考试
])
        |
        v
课程数据处理：
  - 直接使用 API 返回的课程列表
        |
        v
作业数据处理：
  - 建立 courseId -> courseName 映射
  - 为缺少课程名的作业补充课程名
  - 按 lp_homework_id 去重（保留状态优先级更高的版本）
        |
        v
考试数据处理：
  - 建立 courseId -> courseName 映射
  - 为缺少课程名的考试补充课程名
        |
        v
返回 { success: true, courses, homeworks, exams, message }
```

**作业去重策略**：

使用 `Map` 数据结构，以 `lp_homework_id` 或 `homework_name` 为键。当遇到重复项时，保留状态优先级更高的版本：

```javascript
const statusPriority = {
  completed: 5,    // 已完成（最高）
  peer_review: 4,  // 待互评
  in_progress: 3,  // 进行中
  overdue: 2,      // 已逾期
  pending: 1       // 待处理（最低）
};
```

**主进程数据合并**（`realSync()` 函数）：

同步完成后，主进程将 API 返回的数据映射为本地数据结构：
1. 为每门课程分配递增的本地 `id`，同时保留学习通原始 `lp_course_id`
2. 建立 `lp_course_id -> 本地 id` 的映射表
3. 为每条作业/考试分配递增的本地 `id`，通过映射表关联到本地课程 `id`
4. 更新 `appData.currentId` 计数器
5. 记录同步时间到 `appData.chaoxing.lastSync`
6. 调用 `saveData()` 持久化

## 7. 错误处理设计

### 7.1 网络请求错误处理

**chaoxing-api.js 模块**：

所有 API 函数均采用 `try-catch` 包裹，捕获网络异常后返回统一错误格式：

```javascript
// 标准错误返回格式
{ success: false, message: '错误描述信息' }
```

**具体错误场景**：

| 场景 | 处理方式 | 错误信息示例 |
|------|----------|-------------|
| Cookie 过期 | 检查 HTML 中是否包含跳转到 passport 的内容 | 'Cookie已过期，请重新登录' |
| 网络超时 | Axios timeout 配置（15 秒）触发异常 | '获取作业列表出错：timeout of 15000ms exceeded' |
| 网络不可达 | catch 捕获网络异常 | '获取课程列表出错：getaddrinfo ENOTFOUND' |
| 响应非字符串 | 检查 `typeof html !== 'string'` | '获取作业页面失败' |
| API 数据异常 | 检查响应中是否缺少预期字段 | '获取课程列表失败，可能Cookie已过期' |

### 7.2 数据存储错误处理

**loadData() 函数**：
- 文件读取异常：捕获 `try-catch`，输出错误日志，使用默认数据模板初始化
- JSON 解析异常：同上处理

**saveData() 函数**：
- 文件写入异常：捕获 `try-catch`，输出错误日志到控制台，不中断应用运行
- 目录不存在：每次保存前调用 `ensureDataDir()` 确保目录存在

### 7.3 IPC 通信错误处理

**渲染进程侧**：
- 所有 `window.api.xxx()` 调用均使用 `try-catch` 包裹
- 调用失败时通过 `showToast()` 向用户显示错误提示
- 关键操作（如登录、同步）设置 loading 状态，在 `finally` 块中重置

**主进程侧**：
- IPC 处理器中通过数据查找结果判断操作是否成功
- 未找到数据时返回 `{ success: false, message: 'xxx不存在' }`

### 7.4 HTML 解析错误处理

**parseHomeworkHTML() 和 parseExamHTML()**：
- 每条记录的解析使用独立 `try-catch`，单条解析失败不影响其他记录
- 解析失败的记录输出错误日志（`console.error`），跳过该条继续处理
- 无法提取必要字段（如 data URL）的 `<li>` 元素直接跳过

### 7.5 用户输入校验

- 登录表单：通过 Element Plus 的 `el-form` 组件进行基础校验
- IPC 参数：主进程对传入参数进行存在性检查（如查找用户、作业、考试时判断是否为 `undefined`）
- Cookie 验证：`validateCookie()` 函数在同步前验证 Cookie 有效性，避免无效请求

## 8. 日志设计

### 8.1 日志输出方式

系统当前采用 `console.log` / `console.error` / `console.warn` 输出日志信息，通过 Electron 主进程的标准输出和渲染进程的 console-message 事件进行日志记录。

### 8.2 主进程日志

主进程通过 Node.js 的 `console` 对象输出日志，日志会显示在启动 Electron 的终端窗口中。

**日志级别与格式**：

| 级别 | 方法 | 使用场景 |
|------|------|----------|
| INFO | `console.log()` | 常规操作日志 |
| ERROR | `console.error()` | 错误信息 |
| WARN | `console.warn()` | 警告信息 |

**日志标签约定**：

| 标签 | 说明 |
|------|------|
| `[Renderer]` | 来自渲染进程的 console 消息 |
| `[LoginWindow]` | 登录窗口相关操作 |
| `[Sync]` | 数据同步相关操作 |
| `[Chaoxing]` | 学习通 API 相关操作 |

**主要日志点**：

```javascript
// 数据加载
console.log('数据已加载');
console.log('已创建默认数据');

// 登录窗口
console.log('[LoginWindow] 导航到:', url);
console.log('[LoginWindow] 检测到登录成功，正在获取Cookie...');
console.log('[LoginWindow] 获取到Cookie:', cookieStr.substring(0, 50) + '...');
console.log('[LoginWindow] Cookie保存成功');
console.error('[LoginWindow] 获取Cookie失败:', e.message);

// 数据同步
console.log('[Sync] 开始同步学习通数据...');
console.log(`[Sync] 同步完成：${courses}门课程，${homeworks}项作业，${exams}场考试`);

// 学习通 API
console.log('[Chaoxing] 作业页面HTML已保存到:', debugPath);
console.log('[Chaoxing] 解析到', homeworks.length, '项作业');
console.log('[Chaoxing] 解析到', exams.length, '场考试');
console.warn('获取作业失败:', message);
console.warn('获取考试失败:', message);

// 错误日志
console.error('加载数据失败:', err);
console.error('保存数据失败:', err);
console.error('解析作业项失败:', e.message);
console.error('解析考试项失败:', e.message);
```

### 8.3 渲染进程日志

渲染进程的日志通过主进程的 `console-message` 事件转发到主进程控制台：

```javascript
mainWindow.webContents.on('console-message', (event, level, message) => {
  console.log(`[Renderer] ${message}`);
});
```

渲染进程中关键日志点：
- 登录流程：`[Login] window.api is undefined`、`[Login] window.api.login is not a function`、`[Login] Error: ...`
- 数据加载失败：`showToast('加载数据失败', 'error')`

### 8.4 调试日志文件

作业页面 HTML 会被保存到系统临时目录，用于开发调试：

```javascript
const debugPath = path.join(require('os').tmpdir(), 'chaoxing_homework_debug.html');
fs.writeFileSync(debugPath, html, 'utf-8');
```

### 8.5 日志改进建议

当前日志系统基于 `console` 输出，适合开发阶段使用。后续可考虑以下改进：
- 引入 `electron-log` 等专业日志库，支持文件日志、日志分级、日志轮转
- 将日志文件存储在 `userData/logs/` 目录下
- 生产环境设置日志级别为 `warn` 以上，减少日志量
- 添加结构化日志字段（时间戳、模块名、操作类型），便于日志分析
