# LearnPass 学习通助手需求分析报告

## 1. 引言

### 1.1 编写目的

本文档是"LearnPass 学习通助手"项目的需求分析报告，旨在对系统的功能需求、非功能需求、用户界面需求、数据需求及约束条件进行全面、系统的分析与描述。本文档的主要读者包括：

- **项目开发团队**：作为系统设计与编码实现的需求依据；
- **测试人员**：作为编写测试用例和验收测试的基准；
- **指导教师**：作为项目评审和进度检查的参考文档；
- **项目管理人员**：作为进度规划、资源分配和风险控制的参考。

本文档将明确系统需要"做什么"，而非"怎么做"，为后续的概要设计和详细设计提供基础。

### 1.2 项目背景

随着高校信息化教学的不断推进，超星学习通（以下简称"学习通"）已成为国内高校广泛使用的在线教学平台之一。学习通承载着课程通知、作业布置、在线考试、成绩查询等核心教学功能。然而，学习通平台在作业和考试管理方面存在以下不足：

- 不同课程的作业、考试、通知分散在各自的课程页面中，学生需要逐个课程查看，耗时且容易遗漏；
- 缺乏统一的跨课程时间线视图，无法一目了然地了解近期所有待办任务的截止时间；
- 缺少灵活的到期提醒机制，学生容易因疏忽而错过截止日期；
- 互评任务与普通作业混在一起，状态跟踪困难。

为解决上述问题，本项目开发"LearnPass 学习通助手"桌面应用程序，通过对接学习通平台API，为高校学生提供一站式的作业与考试管理服务。

## 2. 系统概述

### 2.1 系统目标

本系统旨在开发一款面向高校学生的桌面端学习通作业考试管理工具，具体目标如下：

1. **数据聚合**：自动同步学习通平台上的作业、考试、课程等数据，统一展示在一个界面中，消除信息分散问题；
2. **状态管理**：对作业进行分类管理（待完成、已逾期、已完成、待互评），帮助学生清晰掌握任务进度；
3. **时间管理**：提供日历视图，按日期展示作业和考试安排，支持状态颜色区分，帮助学生合理安排学习时间；
4. **智能提醒**：支持自定义作业和考试截止提醒，避免遗漏重要任务；
5. **数据统计**：提供作业和考试完成情况的可视化统计分析，辅助学生进行学情自查；
6. **桌面体验**：作为原生桌面应用，提供流畅、稳定的用户体验，支持系统托盘常驻。

### 2.2 系统范围

本系统采用 Electron + Vue 3 + Element Plus + Node.js + Axios 技术栈开发跨平台桌面应用，涵盖以下功能模块：

| 模块名称 | 模块说明 |
|---------|---------|
| 用户登录模块 | 支持本地用户登录和学习通账号登录（账号密码/扫码/验证码） |
| 数据同步模块 | 从学习通API获取课程列表、作业列表、考试列表 |
| 作业管理模块 | 作业列表展示、状态筛选、作业详情查看 |
| 考试管理模块 | 考试列表展示、状态筛选、考试详情查看 |
| 课程管理模块 | 课程列表展示 |
| 日历视图模块 | 按日期展示作业和考试安排，支持状态颜色区分 |
| 提醒设置模块 | 作业/考试截止提醒配置与管理 |
| 数据统计模块 | 作业/考试完成情况统计与可视化 |

本系统不涉及以下内容：

- 作业内容的在线编辑或提交功能；
- 考试的在线答题功能；
- 与学习通平台以外的其他教学平台的数据对接；
- 服务端数据存储（所有数据存储在用户本地）。

## 3. 功能需求

### 3.1 用户登录模块

用户登录模块是系统的入口模块，负责用户身份认证和会话管理。系统支持两种登录方式：本地用户登录和学习通账号登录。

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|---------|---------|---------|-------|
| F-1.1.1 | 本地用户登录 | 用户通过本地注册的账号和密码进行登录，系统验证账号密码的正确性后进入主界面 | 高 |
| F-1.1.2 | 本地用户注册 | 新用户可注册本地账号，设置用户名和密码，注册信息存储在本地 | 高 |
| F-1.1.3 | 学习通账号密码登录 | 用户输入学习通的手机号/邮箱和密码，系统通过学习通API进行身份验证，获取登录Cookie | 高 |
| F-1.1.4 | 学习通扫码登录 | 系统展示学习通扫码登录二维码，用户使用学习通APP扫码完成登录，系统获取登录Cookie | 中 |
| F-1.1.5 | 学习通验证码登录 | 用户输入学习通绑定的手机号，系统发送验证码，用户输入验证码完成登录 | 中 |
| F-1.1.6 | Cookie管理 | 系统自动管理学习通登录Cookie的存储、刷新和过期检测，Cookie过期时提示用户重新登录 | 高 |
| F-1.1.7 | 登录状态保持 | 用户关闭应用后再次打开时，自动恢复登录状态，无需重复登录（Cookie有效期内） | 高 |
| F-1.1.8 | 退出登录 | 用户可主动退出登录，清除本地存储的登录凭证和Cookie信息 | 中 |

数据同步模块负责从学习通平台API获取用户的课程、作业和考试数据，并存储到本地供其他模块使用。

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|---------|---------|---------|-------|
| F-1.2.1 | 课程列表同步 | 从学习通API获取用户当前学期所有课程列表，包括课程名称、教师、课程ID等信息 | 高 |
| F-1.2.2 | 作业列表同步 | 从学习通API获取所有课程的作业列表，包括作业标题、截止时间、状态、所属课程等信息 | 高 |
| F-1.2.3 | 考试列表同步 | 从学习通API获取所有课程的考试列表，包括考试标题、开始时间、结束时间、状态、所属课程等信息 | 高 |
| F-1.2.4 | 手动同步触发 | 用户可手动点击同步按钮，触发数据同步操作，同步过程中显示进度提示 | 高 |
| F-1.2.5 | 自动定时同步 | 系统支持配置自动同步间隔（如每30分钟、每1小时），定时从学习通API拉取最新数据 | 中 |
| F-1.2.6 | 同步状态提示 | 数据同步过程中显示加载状态（进度条或加载动画），同步完成后显示同步结果（成功/失败/部分成功） | 高 |
| F-1.2.7 | 同步失败处理 | 当同步失败时（网络异常、Cookie过期、API错误等），显示友好的错误提示信息，并提供重试操作 | 高 |
| F-1.2.8 | 增量同步 | 同步时仅获取上次同步后新增或变更的数据，减少不必要的数据传输 | 低 |
| F-1.2.9 | 本地数据缓存 | 同步获取的数据缓存到本地存储，支持离线查看已同步的数据 | 中 |

## 4. 非功能需求

### 4.1 性能需求

| 需求编号 | 需求名称 | 需求描述 |
|---------|---------|---------|
| NF-4.1.1 | 应用启动时间 | 应用冷启动时间不超过5秒（在主流配置的PC上） |
| NF-4.1.2 | 页面响应时间 | 页面切换和交互操作的响应时间不超过1秒 |
| NF-4.1.3 | 数据同步时间 | 单次完整数据同步（课程+作业+考试）时间不超过10秒（在正常网络条件下） |
| NF-4.1.4 | 内存占用 | 应用正常运行时内存占用不超过300MB |
| NF-4.1.5 | CPU占用 | 应用空闲时CPU占用率不超过2%，数据同步时不超过30% |
| NF-4.1.6 | 列表渲染性能 | 作业/考试列表在数据量不超过500条时，滚动流畅无卡顿 |
| NF-4.1.7 | 离线响应 | 在离线状态下，已缓存数据的页面加载时间不超过2秒 |

### 4.2 安全需求

| 需求编号 | 需求名称 | 需求描述 |
|---------|---------|---------|
| NF-4.2.1 | Cookie安全存储 | 学习通登录Cookie采用加密方式存储在本地，防止明文泄露 |
| NF-4.2.2 | 密码安全存储 | 本地用户密码采用加密哈希算法存储，不存储明文密码 |
| NF-4.2.3 | 数据本地化 | 所有用户数据仅存储在用户本地设备上，不上传至任何第三方服务器 |

## 5. 用户界面需求

- 右侧内容区根据导航选择动态切换对应的功能页面；
- 顶部标题栏显示应用名称、同步状态指示器和用户信息。

- 学习通登录支持账号密码、扫码、验证码三种方式的切换；
- 表单输入框带有清晰的标签和输入提示。

### 5.3 列表页面

- 作业列表和考试列表采用表格或卡片列表形式展示；

### 6.2 数据存储

- 系统采用本地文件存储（JSON文件或轻量级嵌入式数据库）保存用户数据；
- 所有数据存储在用户本地设备的应用数据目录中；
- 数据文件采用合理的目录结构组织，便于管理和备份。

### 6.3 数据流转

1. 用户登录学习通后，系统获取并存储Cookie；
2. 数据同步模块使用Cookie调用学习通API，获取课程、作业、考试数据；
3. 获取的数据经过解析和转换后存储到本地；
4. 各功能模块从本地数据存储中读取数据进行展示；
5. 提醒模块根据本地数据中的截止时间计算提醒触发时间。
