# 学习通作业考试管理助手概要设计说明书

## 1. 引言

### 1.1 编写目的

本文档是"学习通作业考试管理助手"项目的概要设计说明书，旨在对系统的整体架构、模块划分、接口设计、数据结构、安全策略及部署方案进行全面、系统的描述与规划。本文档的主要读者包括：

- **项目开发团队**：作为系统编码实现的架构依据和技术参考；
- **测试人员**：作为编写测试用例、接口测试和集成测试的设计基准；
- **指导教师**：作为项目评审、架构审查和进度检查的参考文档；
- **项目管理人员**：作为技术方案评审、资源分配和风险控制的参考。

本文档在需求分析报告的基础上，进一步明确系统"怎么做"的技术方案，为后续的详细设计和编码实现提供指导。

### 1.2 项目背景

随着高校信息化教学的不断推进，超星学习通已成为国内高校广泛使用的在线教学平台之一。然而，学习通平台在作业和考试管理方面存在信息分散、缺乏统一时间线、缺少到期提醒机制等问题，导致学生经常错过截止日期。

为解决上述问题，本项目开发"学习通作业考试管理助手"桌面应用程序，采用 Electron + Vue 3 + Element Plus + Node.js + Axios 技术栈，通过对接学习通平台API，为高校学生提供一站式的作业与考试管理服务。系统采用 Electron 主进程与渲染进程分离架构，通过 IPC 机制实现进程间安全通信，所有数据本地化存储，保障用户隐私安全。

## 2. 系统架构设计

### 2.1 总体架构

本系统采用经典的 Electron 桌面应用三层架构设计，将系统划分为表示层、业务逻辑层和数据层三个层次。各层之间职责明确、耦合度低，便于独立开发和维护。

#### 2.1.2 架构说明

系统整体遵循"前端展示 -- 桥接通信 -- 后端处理 -- 数据持久化"的数据流转模式：

1. **表示层**负责用户界面的渲染和交互，采用 Vue 3 的响应式数据驱动视图更新，通过 Element Plus 组件库提供标准化的 UI 组件；
2. **预加载桥接层**作为表示层与业务逻辑层之间的安全通信桥梁，通过 Electron 的 `contextBridge` API 将主进程能力安全地暴露给渲染进程；
3. **业务逻辑层**负责核心业务处理，包括窗口管理、IPC 消息处理、数据读写、学习通 API 调用与数据解析；
4. **数据层**采用本地 JSON 文件进行数据持久化存储，所有用户数据保存在用户设备本地。

### 2.3 Electron 架构

#### 2.3.1 进程模型

Electron 采用多进程架构，本系统涉及以下进程：

| 进程类型 | 数量 | 职责 | 对应文件 |
|---------|------|------|---------|
| 主进程 (Main Process) | 1 | 窗口管理、IPC 处理、文件 I/O、API 调用、数据持久化 | main.js, chaoxing-api.js |
| 渲染进程 (Renderer Process) | 1~2 | UI 渲染、用户交互、Vue 3 应用运行 | index.html |
| 预加载脚本 (Preload Script) | 1~2 | IPC 桥接、安全通信通道建立 | preload.js |
| 登录子窗口渲染进程 | 0~1 | 学习通登录页面渲染（临时创建，登录完成后销毁） | 无独立文件（加载远程 URL） |

#### 2.3.2 进程通信机制

系统采用 Electron 的 IPC（Inter-Process Communication）机制实现主进程与渲染进程之间的双向通信：

```
渲染进程 (Vue 3 App)                    主进程 (main.js)
       |                                       |
       |  window.api.xxx(params)               |
       |  (通过 preload.js 桥接)                |
       | -------------------------------------> |
       |  ipcRenderer.invoke('xxx', params)    |
       |                                       |  ipcMain.handle('xxx', handler)
       |                                       |  处理业务逻辑
       |                                       |  读取/写入数据
       |                                       |  调用学习通 API
       |  <------------------------------------- |
       |  返回 Promise 结果                      |
       |                                       |
       |  (主进程主动推送事件)                     |
       |  <------------------------------------- |
       |  webContents.send('event-name', data)  |
       |  ipcRenderer.on('event-name', callback) |
       |                                       |
```

**通信模式说明：**

- **请求-响应模式**（渲染进程 -> 主进程）：渲染进程通过 `ipcRenderer.invoke()` 发起请求，主进程通过 `ipcMain.handle()` 处理请求并返回结果，采用 Promise 异步机制；
- **事件推送模式**（主进程 -> 渲染进程）：主进程通过 `webContents.send()` 主动推送事件，渲染进程通过 `ipcRenderer.on()` 监听事件，用于学习通登录成功通知等场景。

#### 2.3.3 安全隔离策略

系统采用 Electron 推荐的安全最佳实践：

| 安全配置项 | 配置值 | 说明 |
|-----------|-------|------|
| nodeIntegration | false | 禁止渲染进程直接访问 Node.js API |
| contextIsolation | true | 启用上下文隔离，隔离预加载脚本与页面 JS 运行环境 |
| enableRemoteModule | false | 禁用远程模块，防止远程代码执行风险 |
| sandbox | false | 关闭沙箱（因预加载脚本需要 Node.js 能力） |
| preload 脚本 | preload.js | 通过 contextBridge 暴露受限 API 接口 |

| M-3.2.2 | IPC 通道映射 | 将渲染进程的函数调用映射到对应的 ipcRenderer.invoke 通道 |

API 封装模块通过关键词匹配和时间比较实现作业状态的智能判断，判断优先级如下：

| 优先级 | 匹配关键词 | 判断结果 | 提交状态 |
|-------|-----------|---------|---------|
| 1 | 已批阅、已提交、已完成、待批阅、已互评 | completed | submitted |
| 2 | 待互评 | peer_review | submitted |
| 3 | 已截止、已过期、逾期、待重做 | overdue | not_submitted |
| 4 | 未交、未提交（有剩余时间） | pending | not_submitted |
| 5 | 未交、未提交（无剩余时间） | overdue | not_submitted |
| 6 | 进行中 | in_progress | not_submitted |
| 7 | 截止时间已过（自动判断） | overdue | not_submitted |
| 8 | 默认 | pending | not_submitted |

#### 3.4.5 数据去重策略

同步获取的作业数据可能存在重复项，系统采用以下去重策略：

- **去重键**：以 `lp_homework_id`（学习通作业 ID）或 `homework_name`（作业名称）作为去重键；
- **优先级规则**：当出现重复项时，保留状态优先级更高的版本。状态优先级为：completed(5) > peer_review(4) > in_progress(3) > overdue(2) > pending(1)；
- **实现方式**：使用 JavaScript Map 数据结构，以去重键为 key，遍历时比较并保留高优先级版本。

## 4. 数据库设计

### 4.1 数据存储方案

#### 4.1.1 存储方案选型

本系统采用本地 JSON 文件作为数据存储方案，原因如下：

| 对比维度 | JSON 文件 | SQLite 数据库 |
|---------|----------|-------------|
| 实现复杂度 | 低（Node.js 内置 fs 模块） | 中（需引入 sqlite3 依赖） |
| 数据量 | 适合（预计不超过 1000 条记录） | 适合大规模数据 |
| 查询能力 | 需全量加载后内存筛选 | 支持 SQL 查询 |
| 部署便利性 | 无额外依赖 | 需编译原生模块 |
| 数据可读性 | 高（可直接打开查看） | 低（二进制格式） |
| 备份恢复 | 简单（复制文件） | 需导出工具 |

考虑到本系统为桌面单用户应用，数据量有限，JSON 文件方案在实现复杂度、部署便利性和数据可读性方面具有明显优势。

#### 4.1.2 存储策略

- **读写策略**：采用"全量读取 + 增量写入"策略，应用启动时全量加载数据到内存（appData 全局变量），数据变更时全量写入文件；
- **文件格式**：JSON 格式，使用 2 空格缩进，UTF-8 编码；
- **并发控制**：单进程运行，不存在并发写入问题；
- **数据备份**：用户可通过数据导出功能手动备份数据。

- **成功响应**：返回包含 `channelList` 或 `courseList` 字段的 JSON 数据
- **失败响应**：重定向到 `passport2.chaoxing.com`（表示 Cookie 已过期）

- **URL**：`GET https://mooc1-api.chaoxing.com/mycourse/backclazzdata?view=json&mcode=`

- **响应结构**：

#### 5.2.6 作业列表接口详情

- **URL**：`GET https://mooc1-api.chaoxing.com/work/stu-work`
- **认证方式**：通过请求头中的 Cookie 字段
- **返回格式**：HTML 页面（非 JSON），需通过 HTML 解析提取数据
- **解析策略**：
  1. 使用正则表达式匹配所有 `<li>` 标签；
  2. 从 `data` 属性或 `href` 属性中提取包含 `mooc-ans` 或 `taskrefId` 的 URL；
  3. 从 URL 参数中提取作业 ID（taskrefId）和课程 ID（courseId）；
  4. 从 `<p>` 标签提取作业名称，从 `aria-label` 属性提取完整描述；
  5. 从状态关键词判断作业状态。

#### 5.2.7 考试列表接口详情

- **URL**：`GET https://mooc1-api.chaoxing.com/exam-ans/exam/phone/examcode`
- **认证方式**：通过请求头中的 Cookie 字段
- **返回格式**：HTML 页面（非 JSON），需通过 HTML 解析提取数据

  3. 从 `<dl class="ks_text">` 内的 `<dt>` 提取考试名称，`<dd>` 提取时间信息；

## 6. 安全设计

### 6.1 进程安全隔离

系统采用 Electron 推荐的多层安全隔离策略，确保渲染进程无法直接访问系统资源：

| 安全措施 | 实现方式 | 防护目标 |
|---------|---------|---------|
| 禁用 Node.js 集成 | nodeIntegration: false | 防止渲染进程直接使用 Node.js API（如 fs、child_process） |
| 启用上下文隔离 | contextIsolation: true | 隔离预加载脚本与页面 JavaScript 的全局对象 |
| 禁用远程模块 | enableRemoteModule: false | 防止通过 remote 模块访问主进程资源 |
| 最小权限预加载 | 仅暴露必要的 IPC 接口 | 限制渲染进程可调用的功能范围 |

### 6.2 IPC 通信安全

| 安全措施 | 实现方式 | 说明 |
|---------|---------|------|
| 白名单机制 | 仅通过 contextBridge 暴露预定义的 API 接口 | 渲染进程只能调用已注册的 IPC 通道 |
| 参数校验 | 主进程对 IPC 请求参数进行类型和合法性校验 | 防止恶意参数导致异常行为 |
| 错误隔离 | IPC 处理函数使用 try-catch 包裹 | 防止处理异常影响主进程稳定性 |
| Cookie 脱敏 | getChaoxingStatus 接口返回的 Cookie 字段以 "***" 替代 | 防止 Cookie 通过 IPC 泄露到渲染进程 |

### 6.3 数据安全

| 安全措施 | 实现方式 | 说明 |
|---------|---------|------|
| 本地存储 | 所有数据存储在用户本地设备 | 数据不上传至任何第三方服务器 |
| 数据目录隔离 | 使用 Electron 的 userData 目录 | 数据存储在操作系统标准应用数据目录中 |
| 文件编码 | UTF-8 编码 | 确保中文字符正确存储和读取 |
| 错误恢复 | 数据加载失败时使用默认数据恢复 | 防止数据损坏导致应用无法启动 |

### 6.4 网络通信安全

| 安全措施 | 实现方式 | 说明 |
|---------|---------|------|
| HTTPS 协议 | 所有学习通 API 请求均使用 HTTPS | 确保数据传输加密 |
| 请求超时 | 设置 15 秒请求超时 | 防止请求挂起影响应用响应 |
| 状态码校验 | validateStatus: status < 500 | 对 4xx 错误进行业务处理而非抛出异常 |
| Cookie 过期检测 | 同步前先验证 Cookie 有效性 | 避免使用过期 Cookie 发送无效请求 |

### 6.5 学习通登录安全

| 安全措施 | 实现方式 | 说明 |
|---------|---------|------|
| 独立 Session | 登录窗口使用独立 session | 避免登录过程污染主窗口的 session |
| 自动关闭 | 登录成功后自动关闭登录窗口 | 减少 Cookie 暴露时间 |
| 登录检测 | 通过 URL 跳转检测登录是否成功 | 仅在跳转到学习通主域时才获取 Cookie |
| Cookie 存储 | Cookie 存储在本地 JSON 文件中 | 不通过网络传输 Cookie |
